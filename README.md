# maternal-health-n8n-prototype

An advanced, multi-agent automated triage and hyper-localized storytelling pipeline built on n8n. This system utilizes engineered organizational and risk-mitigation frameworks to optimize maternal health communication over feature phone SMS gateways.

## 📐 Core Architecture Frameworks

The system's decision-making logic, routing protocols, and safety rails are calibrated across three foundational pillars:

### 1. RANK Calibration (Infrastructure Hierarchy)
* **RANK 1: Core Triage** – Governs initial ingestion and classification of incoming symptoms.
* **RANK 2: Sequential Routing & Filtering** – Progresses user data through specific gestational milestones and geographic parameters.
* **RANK 3: Hyper-Localization & Terminal Sink** – Handles dynamic, culturally component narrative response generation and system audits.

### 2. HUNT Handoff Protocols (Agent Communication)
* **H (Handoff Triggers):** Automatically pivots from routine flows to emergency tracks upon critical intent classification.
* **U (Unified Context):** Persists session parameters (trimester, region, history) seamlessly across processing gates.
* **N (Negotiation Rules):** Implements automated fallback hierarchies to handle missing data fields without failing.
* **T (Termination Conditions):** Gracefully concludes execution cycles when the outbound gateway registers a clean `200 OK` network transmission response.

### 3. GUARD Safety Rails (Risk Mitigation)
* **GUARD 1: Health Intake Shield** – Sanitizes, formats, and structuralizes raw string inputs at the message ingestion gate.
* **GUARD 2: Health Filter Shield** – Enforces validation metrics on internal parameters to block logic or routing bypasses.
* **GUARD 3: Continuous Asynchronous Audit Shield** – Anomaly tracking layer that flags drifts in baseline generation >15%, creates immutable records of reasoning chains, and handles red-team verification routines.

## 🚀 How to Import and Deploy on n8n

To view, test, or modify this technical prototype inside your own instance:

1. Download the raw `My workflow.json` file from this repository.
2. Open your live **n8n canvas workspace**.
3. Click the three-dot menu icon (`...`) in the top right-hand corner.
4. Select **Import from file** and upload the downloaded `.json` blueprint.
5. Double-click the first **Webhook Trigger Node**, switch the URL view to **Production URL**, and toggle the workflow switch to **Active**.
